# 🛠 Техническая документация проекта "Джерри и сырное воздаяние"

## 🏗 Архитектура

Игра построена на базе модульной архитектуры с четким разделением ответственности:

- **Core (Ядро)**: Базовые системы (ресурсы, звук, камера, машина состояний).
- **Entities (Сущности)**: Игровые объекты с собственной логикой и физикой.
- **Scenes (Сцены)**: Изолированные экраны игры (Меню, Уровни, Титры).
- **UI (Интерфейс)**: Элементы пользовательского взаимодействия.

### 🔄 Машина состояний (State Machine)

Центральный механизм управления игровым процессом. Класс `StateMachine` в `core/state_machine.py` управляет переходами между сценами. Каждая сцена наследуется от `BaseState` и реализует методы `enter`, `exit`, `handle_events`, `update` и `draw`.

### 🎥 Система камеры

Реализована в `core/camera.py`. Камера использует метод **Lerp** (линейная интерполяция) для плавного следования за игроком. Добавлено динамическое смещение в сторону курсора мыши, что позволяет игроку видеть больше пространства в направлении взгляда.

### 📦 Менеджер ресурсов (Resource Manager)

Класс `ResourceManager` в `core/resource.py` обеспечивает:

- Централизованную загрузку и кэширование изображений, шрифтов и звуков.
- Экономию памяти за счет повторного использования уже загруженных ассетов.
- Удобный доступ к ресурсам через строковые константы из `constant.py`.

## 🎮 Игровые механики

### Физика и коллизии

Используется прямоугольная коллизия Pygame (`colliderect`). Физика игрока включает гравитацию, трение и расчет дельты времени (`dt`) для независимости от частоты кадров.

### Искусственный интеллект врагов

Враги (`Tom`, `Broom`) обладают простым состоянием патрулирования. Они реагируют на **Приманку** (`Decoy`), если она находится в радиусе их видимости, переключаясь на преследование приманки.

### Битва с Боссом (BossTom)

Босс имеет три фазы:

1. **Стрельба ракетами**: Запуск самонаводящихся снарядов в позицию игрока.
2. **Сброс ящиков**: Создание препятствий и попытка раздавить игрока.
3. **Рывок**: Быстрое перемещение по арене.

## 🎬 Мультимедиа и Видео

Для воспроизведения финального видео используется комбинация **OpenCV** и **MoviePy**:

- `cv2.VideoCapture` обеспечивает чтение кадров.
- `MoviePy` извлекает аудиодорожку во временный файл `temp_credits_audio.mp3`.
- Синхронизация видео и звука осуществляется программно через расчет `target_frame` на основе времени воспроизведения аудио.

## 📦 Сборка (EXE)

Для создания исполняемого файла планируется использование **PyInstaller**.
Основные параметры сборки:

- `--onefile`: Объединение всех скриптов в один EXE.
- `--noconsole`: Отключение консольного окна при запуске.
- `--add-data`: Включение папки `asset` и `level` в сборку.
- `--icon`: Использование `asset/other/icon.ico`.

*Инструкция по сборке будет добавлена после финального тестирования.*
